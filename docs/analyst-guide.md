# Analyst Guide

This guide is for BLV and BAG analysts who use the SENTINEL dashboard for daily public health intelligence work.

---

## Daily Workflow

A recommended daily workflow for reviewing SENTINEL data:

### 1. Open the Command Center

Start your morning by opening the SENTINEL dashboard. The **Command Center** is the default landing page and provides an immediate overview:

- **KPI Cards** at the top show today's totals: events collected, critical/high-risk count, average Swiss relevance
- **Risk Heatmap** highlights geographic hotspots
- **Priority Events** lists the highest-scoring events requiring attention
- **Trend Sparklines** show 7-day movement in event volume and risk levels

**What to look for:** Any increase in critical/high-risk events. New countries appearing in the heatmap. Upward risk trends.

### 2. Review the Triage Queue

Navigate to the **Triage Queue** to process new events:

- Events are displayed as cards sorted by Swiss relevance (most relevant first)
- Each card shows: disease, countries, risk score, Swiss relevance, source, One Health tags
- Cards are color-coded by risk category: red (critical), orange (high), yellow (medium), blue (low)

**For each event in the queue:**

1. Read the title and summary
2. Check the risk score and Swiss relevance
3. If LLM analysis is available, expand the analysis section for the detailed narrative
4. Take action:
   - **Acknowledge** -- Mark as reviewed, set status to MONITORING
   - **Escalate** -- Raise to ESCALATED status, assign priority
   - **Link** -- Connect to an existing Situation if it relates to an ongoing outbreak
   - **Annotate** -- Add your assessment, notes, or recommended actions
   - **Override** -- Adjust the risk score if you disagree with the automated assessment

### 3. Manage Situations

Navigate to the **Situations** view. This is a Kanban-style board with columns:

- **ACTIVE** -- Outbreaks currently being monitored
- **WATCH** -- Lower-priority situations worth tracking
- **ESCALATED** -- Situations requiring leadership attention or cross-agency coordination
- **RESOLVED** -- Contained or concluded outbreaks

**For each active situation:**

1. Check if new events have been linked (the event count on the card updates automatically)
2. Review the timeline for recent developments
3. Update the summary if the situation has evolved
4. Adjust priority (P1--P4) based on current assessment
5. Update the Swiss impact assessment and recommended actions

### 4. Export and Report

At the end of your review, use the **Export Center** to generate reports:

- **Daily Brief** -- Markdown summary of today's intelligence (auto-generated by the pipeline)
- **Filtered Export** -- CSV or JSON of events matching your criteria
- **Situation Report** -- Deep-dive on a specific situation with full annotation history

## Understanding Risk Scores

### Risk Score (0--10)

This measures global danger. It is calculated by the rule engine based on:

| Factor | Max Points |
|--------|------------|
| Geographic proximity to Switzerland | 4.0 |
| Disease on the high-concern list | 2.5 |
| Zoonotic disease (One Health) | 1.0 |
| Reported deaths | 2.0 |
| Source authority (ECDC, WHO DON) | 1.0 |

Events scoring >= 4.0 also receive LLM analysis from Claude, which may adjust the score up or down based on epidemiological context.

### Swiss Relevance (0--10)

This measures how directly the event affects Switzerland:

- **10.0** -- Event is in Switzerland
- **5.0+** -- Event in a border country (DE, FR, IT, AT, LI)
- **2.5+** -- Event in a European country
- **Additional points** for Swiss vector diseases, zoonotic/foodborne tags, high case counts

### Risk Categories

| Category | Score | Dashboard Color | What It Means |
|----------|-------|-----------------|---------------|
| CRITICAL | 8--10 | Red | Immediate threat, requires action today |
| HIGH | 6--8 | Orange | Significant concern, prioritize for review |
| MEDIUM | 4--6 | Yellow | Monitor, check developments |
| LOW | 0--4 | Blue | Logged for awareness, no action needed |

### When to Override a Score

You should consider overriding the automated score when:

- You have information not available in the automated data (e.g., intelligence from partner agencies)
- The disease context has changed (e.g., a new vaccine makes a threat less urgent)
- The automated system over/underweights a specific factor for this event
- A historical pattern suggests different risk than the current data indicates

Document your rationale in the annotation when overriding.

## Views Reference

### Command Center

The operational overview. Designed for a quick morning scan.

- **KPI Cards** -- Summary statistics with sparkline trends
- **Risk Heatmap** -- Geographic distribution of events by risk level
- **Priority Feed** -- Top events sorted by risk score
- **Source Summary** -- Event counts per data source

### Global Map

Interactive map showing event locations worldwide.

- Markers are sized by case count and colored by risk category
- Click a marker to see event details
- Toggle layers: events, trade routes, migration routes, vector habitats
- Use the time slider to animate the spread of events over time

Requires a Mapbox token for full functionality.

### Triage Queue

The primary analyst work surface.

- **Filter Bar** -- Filter by source, disease, risk category, country, Swiss relevance threshold
- **Search** -- Full-text search across event titles and summaries
- **Sort** -- By risk score, Swiss relevance, date, or source
- **Batch Operations** -- Select multiple events to assign status, link to situation, or export
- **Quick Actions** on each card: assign status, link, annotate, override score

### Situations

Track evolving outbreaks that span multiple events and sources.

- **Creating a Situation:** Click "New Situation" and provide title, diseases, countries, lead analyst, priority, and initial summary
- **Linking Events:** From the triage queue, select events and link them to an existing situation
- **Kanban Board:** Drag situations between status columns (ACTIVE, WATCH, ESCALATED, RESOLVED, ARCHIVED)
- **Situation Detail Page:** Timeline of linked events, annotation thread, One Health impact matrix (human/animal/environmental status), Swiss impact assessment

**Priority Levels:**

| Priority | Meaning |
|----------|---------|
| P1 | Immediate action required |
| P2 | Significant, requires attention within 24 hours |
| P3 | Important, requires attention within the week |
| P4 | Watch, periodic review |

### Analytics

Data analysis and trend visualization.

- **Disease Trends** -- Events per day grouped by disease (line chart)
- **Source Comparison** -- Events per source per day (stacked bar chart)
- **Risk Timeline** -- Average risk score and Swiss relevance over time
- **Geographic Spread** -- How events spread across regions over time

### Watchlists

Custom alert criteria. When new events match your watchlist criteria, they are highlighted in the triage queue.

- **Create a Watchlist:** Define criteria by diseases, countries, minimum risk score, and One Health tags
- **Pre-built Templates:** Swiss border countries, PHEIC diseases, BLV priority zoonoses
- **Edit/Delete:** Adjust criteria as your monitoring priorities change

### Export Center

Generate reports and data exports.

- **CSV Export** -- Tabular data for spreadsheet analysis (filterable by all event fields)
- **JSON Export** -- Machine-readable data for system integration
- **Daily Brief** -- Markdown intelligence summary (auto-generated each pipeline run)
- **Custom Filters** -- Export only events matching specific criteria (date range, source, disease, risk category, country, Swiss relevance threshold)

## Annotations

Annotations are the primary way analysts add institutional knowledge to SENTINEL data.

### Annotation Types

| Type | Use Case |
|------|----------|
| ASSESSMENT | Your professional evaluation of the event's risk |
| NOTE | General observations, context, or background information |
| ACTION | Recommended or taken actions (e.g., "Notified cantonal authorities") |
| LINK | References to external documents or related events |
| ESCALATION | Record of escalation with justification |

### Visibility Levels

| Level | Who Can See |
|-------|-------------|
| INTERNAL | Your agency only (BLV or BAG) |
| SHARED | Both agencies |
| CONFIDENTIAL | Restricted to named individuals |

### Best Practices

- Always annotate when you override a risk score (document your reasoning)
- Use ASSESSMENT type for substantive evaluations
- Use ACTION type to create an audit trail of responses
- Link related events to maintain institutional memory
- Keep annotations concise but complete enough for a colleague to understand

## Multi-Agency Coordination

### Agency-Specific Views

When you select your agency (BLV or BAG), the dashboard adjusts:

- **BLV view** prioritizes: zoonotic events, animal health, foodborne outbreaks, vector-borne diseases. Priority sources: WOAH, ECDC, ProMED.
- **BAG view** prioritizes: human health events, pandemic signals, respiratory diseases, vaccine-preventable diseases. Priority sources: WHO DON, ECDC, WHO EIOS.
- **Joint view** shows One Health intersection events relevant to both agencies.

### Cross-Agency Situations

Situations can be shared between agencies. When a zoonotic spillover event is detected (tagged as `zoonotic` with `species: both`), it is automatically flagged for both BLV and BAG review.

- Each agency can add annotations with appropriate visibility settings
- The situation detail page shows the One Health impact matrix with separate human, animal, and environmental health status fields

## Tips for Effective Use

1. **Start with the Command Center** every morning to get the big picture
2. **Process the triage queue** systematically -- do not skip medium-risk events, they sometimes reveal emerging patterns
3. **Create situations early** -- even if uncertain, a watch-level situation is better than losing track of a developing outbreak
4. **Annotate generously** -- your notes are the institutional memory that makes SENTINEL valuable over time
5. **Use watchlists** for your ongoing concerns -- they save time on filtering and ensure nothing slips through
6. **Review analytics weekly** to spot trends that daily triage might miss
7. **Export and share** -- the daily brief is designed to be forwarded to leadership as-is
